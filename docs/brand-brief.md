# Contexto del proyecto — Vorello Agency

## 1. Qué es Vorello

Vorello Agency es una agencia digital especializada en crear experiencias web, productos digitales y ecommerce de alto nivel.

La marca busca transmitir:
- calidad
- criterio
- diseño cuidado
- tecnología moderna
- atención al detalle
- claridad en el proceso
- enfoque premium

Vorello no debe sentirse como una agencia genérica de “hacemos webs”, sino como un estudio digital que diseña y desarrolla sistemas digitales bien pensados, visualmente cuidados y técnicamente sólidos.

## 2. Posicionamiento

Vorello apunta a clientes que valoran el diseño, la tecnología y la calidad como inversión estratégica.

No compite por precio.
No busca volumen.
No acepta proyectos improvisados o sin objetivos claros.

La web debe comunicar que Vorello trabaja con proceso, estrategia, UX/UI, desarrollo moderno y estándares altos.

## 3. Frase principal de marca

Usar como base conceptual:

“Diseñamos y desarrollamos productos digitales bien pensados, visualmente cuidados y técnicamente sólidos.”

Esta frase representa muy bien el ADN de Vorello y debe guiar el tono de la landing.

## 4. Hero recomendado

El hero debe ser fuerte, minimalista y premium.

Headline principal sugerido:

“Diseño. Tecnología. Producto.”

Subheadline:

“Diseñamos y desarrollamos productos digitales bien pensados, visualmente cuidados y técnicamente sólidos.”

CTAs posibles:
- “Iniciar proyecto”
- “Ver cómo trabajamos”
- “Explorar servicios”
- “Trabajar con nosotros”

Evitar frases genéricas como:
- “Transformamos ideas en realidad”
- “Impulsamos tu negocio”
- “Creamos soluciones digitales”
- “Tu agencia digital de confianza”

## 5. Qué hace Vorello

### Experiencias web de alto rendimiento

Sitios web modernos orientados a conversión, performance, SEO técnico, accesibilidad y calidad visual.

Incluye:
- sitios corporativos premium
- landing pages
- CRO
- SEO técnico
- accesibilidad
- Core Web Vitals

### Productos digitales a medida

Web apps, plataformas internas, paneles de gestión, portales de clientes, sistemas a medida e integraciones.

Incluye:
- web apps
- dashboards
- sistemas internos
- portales privados
- automatizaciones
- integraciones entre sistemas

### Ecommerce de alto nivel

Experiencias ecommerce modernas, escalables y orientadas a conversión.

Incluye:
- ecommerce custom
- arquitectura headless
- UX optimizada
- performance
- SEO técnico

## 6. Qué NO hace Vorello

Vorello no ofrece como servicio principal:
- gestión de redes sociales
- creación de contenido para redes
- campañas publicitarias Meta Ads / Google Ads
- branding completo desde cero
- diseño gráfico aislado
- piezas sueltas sin contexto de producto
- webs rápidas sin estrategia
- proyectos urgentes sin planificación

Esto debe influir en el copy: la web no debe vender “marketing general”, sino diseño y desarrollo digital premium.

## 7. Diferencial

El diferencial central de Vorello:

“No construimos páginas aisladas, sino sistemas digitales pensados para funcionar, escalar y generar resultados reales.”

Este concepto puede aparecer en una sección tipo statement o manifiesto.

## 8. Proceso de trabajo

La web debe reflejar que Vorello trabaja con un proceso claro:

1. Descubrimiento
2. Estrategia y definición
3. UX/UI
4. Desarrollo
5. Optimización y validación
6. Entrega y lanzamiento
7. Evolución

Cada sección de la web debería reforzar la idea de orden, claridad y calidad.

## 9. Personalidad visual

Estética deseada:
- premium
- minimalista
- oscura
- tecnológica
- elegante
- moderna
- similar en sensibilidad a Vercel, Linear, Stripe, pero con identidad propia

Evitar:
- diseños coloridos genéricos
- exceso de ilustraciones stock
- estilo startup cliché
- gradients exagerados
- secciones saturadas
- copy demasiado vendedor

## 10. Paleta de colores

Base:
- Carbon Black: #0D0F11
- Graphite Metal: #1A1D21
- Steel Grey: #2A2E33

Acentos:
- Neon Blue: #2D8FFF
- Vorello Electric Violet: #7B4CFF

Chrome / tonos metálicos:
- Chrome Highlight: #E3E7EE
- Chrome Deep: #5A6270

La interfaz debe usar principalmente negros, grises, blanco suave y acentos violetas/azules de forma sutil.

## 11. Tipografía

Usar una combinación moderna y premium.

Preferencia:
- Headings: Bebas Neue o alternativa fuerte/condensada si encaja
- Body: Space Grotesk o tipografía sans moderna

La tipografía debe sentirse tecnológica, limpia y editorial.

## 12. Secciones recomendadas para la landing

Estructura sugerida:

1. Navbar
2. Hero
3. Statement / manifiesto breve
4. Servicios
5. Cómo trabajamos
6. Diferencial / por qué Vorello
7. Clientes ideales / fit
8. Tecnologías o enfoque técnico
9. CTA final
10. Footer

No sobrecargar la landing inicial. Debe ser clara, directa y visualmente muy cuidada.

## 13. Componentes recomendados

Crear componentes reutilizables y bien organizados:

- Container
- Section
- SectionHeading
- Button
- Badge
- ServiceCard
- ProcessStep
- StatementBlock
- CTASection
- Navbar
- Footer

Pensar el proyecto como un sistema de diseño, no como una página armada de forma aislada.

## 14. Storybook

Storybook puede ser útil para Vorello porque la marca necesita consistencia visual y escalabilidad.

Si se usa Storybook, documentar:
- botones
- cards
- inputs
- badges
- headings
- secciones del marketing site
- componentes futuros del portal privado

Storybook debe funcionar como laboratorio de UI y documentación viva del sistema visual.

## 15. Portal privado futuro

La web pública debe poder evolucionar en el futuro hacia un portal privado para clientes.

No construirlo ahora como prioridad, pero sí considerar que más adelante podría existir:

- login
- portal de cliente
- estado del proyecto
- entregables
- pendientes del cliente
- documentos
- facturación básica
- soporte

El concepto no debe venderse como “software complejo”, sino como:

“Experiencia de proyecto centralizada.”

## 16. Stack recomendado

Stack principal para la web:

- Next.js
- React
- TypeScript
- Tailwind CSS
- Vercel
- Figma como fuente de diseño

Posibles futuras integraciones:
- Firebase Auth
- Firestore
- Firebase Storage
- Resend
- CMS headless si se requiere

## 17. Criterios técnicos

El código debe priorizar:
- componentes reutilizables
- estructura clara
- accesibilidad
- performance
- responsive design
- buena semántica HTML
- escalabilidad
- estilos consistentes
- separación entre layout, sections y UI components

Evitar:
- componentes gigantes
- lógica mezclada con presentación
- estilos duplicados
- valores hardcodeados sin criterio
- animaciones excesivas
- dependencias innecesarias

## 18. Tono de comunicación

El copy debe sonar:
- claro
- seguro
- premium
- estratégico
- directo
- sin exageraciones

No usar un tono demasiado corporativo ni demasiado informal.

Ejemplo de tono correcto:

“Creamos productos digitales con criterio, diseño cuidado y tecnología sólida.”

Ejemplo de tono incorrecto:

“Llevamos tu empresa al siguiente nivel con soluciones innovadoras y disruptivas.”

## 19. Objetivo actual de la web

La prioridad actual de la web es:

1. Elevar la percepción de marca
2. Explicar qué hace Vorello
3. Mostrar proceso y criterio
4. Filtrar clientes no ideales
5. Generar confianza
6. Captar oportunidades de proyectos premium

La web no debe intentar explicar todo. Debe abrir la puerta a una conversación comercial.

## 20. Principio rector

Cada decisión de diseño, copy y desarrollo debe responder a esta idea:

Vorello debe sentirse como una agencia seria, premium y técnicamente sólida, no como una plantilla genérica de agencia digital.
