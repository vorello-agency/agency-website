# Implementation Roadmap

## Objetivo

Entregar la web pública de Vorello con calidad premium, stack Next.js + TypeScript + Tailwind, y motion GSAP-first.

## Fase 1 — Base del sistema (MVP técnico)

### Entregables

- Layout base (`Navbar`, `Footer`, `Container`, `Section`, `SectionHeading`).
- Tokens iniciales aplicados (color, tipografía, spacing, radius).
- Home estructurada con secciones definidas del brief.
- Setup de motion base con GSAP (`register`, utilidades, reveals básicos).

### Criterio de cierre

- Build, lint y typecheck en verde.
- Responsive sin overflow horizontal.
- Estructura de componentes reusable.

## Fase 2 — Secciones clave (MVP visual)

### Entregables

- Hero premium con entrada cuidada.
- Servicios con cards consistentes y estados hover/focus.
- Proceso con narrativa clara (scroll-driven opcional según calidad/performance).
- Statement/Diferencial con jerarquía editorial.
- CTA final alineado al tono de marca.

### Criterio de cierre

- Dirección visual consistente con `docs/brand-brief.md`.
- Motion con propósito (sin sobreanimación).
- A11y base cubierta en secciones principales.

## Fase 3 — Pulido pre-lanzamiento

### Entregables

- Ajustes finos de copy y jerarquía.
- Revisión de performance (LCP/CLS/INP) y optimización de assets.
- Revisión SEO on-page base (metadata principal).
- QA cross-device (mobile/tablet/desktop).

### Criterio de cierre

- Checklist de release completo.
- Sin errores críticos visuales, funcionales o de accesibilidad.

## Prioridad de ejecución sugerida

1. Sistema base de componentes + tokens.
2. Hero.
3. Servicios.
4. Proceso.
5. Statement/Diferencial.
6. CTA final + footer polish.
